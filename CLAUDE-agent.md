## Deployment (this agent is Python, NOT Next.js)
This app does **not** use `npm install` / `npm run build`. If the deploy webhook
is hard-wired to the Next.js build, point it at the Python flow below (or run
these by hand once on the VM).

**One-time VM setup** (after the first clone, and whenever Python deps change):
```
bash deploy/bootstrap.sh          # creates .venv, pip install -e ., alembic upgrade head
pm2 start ecosystem.config.js && pm2 save
```
`bootstrap.sh` needs a populated repo-local **`.env`** (copy from
`.env.prod.example`) — PM2 loads it and injects it into both processes.

**On each push to `main`** the webhook only needs:
```
git pull && pm2 reload ecosystem.config.js --update-env
```

**Processes** (see `ecosystem.config.js`): `switchboard-web` (uvicorn, the console
+ `/api/*` + `/run`) and `switchboard-scheduler` (APScheduler loop). Served under
Caddy at `/agents/adam-abundance/`; set `APP_BASE_PATH=/agents/adam-abundance`
in `.env` (links + redirects are prefixed centrally — no per-template changes).

**Health/verify:** `GET /api/health`, or `SWITCHBOARD_ENV_FILE=.env .venv/bin/switchboard selfcheck`.

**Starts locked:** `SWITCHBOARD_KILL_SWITCH=1` (observe + plan only, no live
actions). Flip to `0` per-action when ready. Spend caps are off
(`SPEND_CAPS_ENABLED=0`); Slack is intentionally not wired.
